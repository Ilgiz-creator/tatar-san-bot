# tatar-san-bot

Телеграм-бот на Python, связанный с татарским языком (упражнения, фразы, словарные подсказки и т.п.), который запускается на вашем сервере и обрабатывает сообщения пользователей напрямую через Telegram.

> Репозиторий: <https://github.com/Ilgiz-creator/tatar-san-bot>

---

## Возможности

- Общение с ботом в Telegram (команды и обычные сообщения).
- Логика бота вынесена в отдельный модуль `bot/`, что упрощает доработку.
- Отдельная директория `tests/` для автоматических тестов основной логики.
- Зависимости описаны в `requirements.txt`, проект легко развернуть на любом сервере с Python.

---

## Требования

- **Python** 3.8+ (рекомендуется 3.10+).
- **git** для клонирования репозитория.
- Аккаунт в Telegram и созданный бот через `@BotFather` (нужен токен).
- (Опционально) ключ OpenAI API, если вы используете интеграцию с моделями OpenAI.

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Ilgiz-creator/tatar-san-bot.git
cd tatar-san-bot
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python3 -m venv venv
source venv/bin/activate  # Linux / macOS

# В Windows (PowerShell):
# python -m venv venv
# .\venv\Scripts\activate

pip install --upgrade pip
pip install -r requirements.txt
```

### 3. Настроить переменные окружения

Создайте файл `.env` в корне проекта (рядом с `requirements.txt`) или выставьте переменные окружения любым удобным способом.

Пример `.env`:

```env
TELEGRAM_BOT_TOKEN=1234567890:ABCDEF_your_token_here
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Если ключ OpenAI не используется в вашей конфигурации, переменную `OPENAI_API_KEY` можно не указывать.

> Названия переменных могут отличаться в зависимости от того, как именно вы реализовали конфигурацию в коде — при необходимости загляните в модуль с настройками внутри директории `bot/`.

### 4. Запуск бота

Наиболее типичные варианты запуска (выберите тот, который соответствует структуре вашего кода):

```bash
# если в пакете bot настроен точка входа через __main__.py
python -m bot

# или, если основная логика лежит в файле main.py
python bot/main.py
```

После запуска бот начнёт опрашивать Telegram (long polling) или обрабатывать запросы, если вы используете вебхуки (в зависимости от реализации в коде).

## Структура проекта

Минимальная структура репозитория:

```text
.
├── bot/              # Основной код бота (хендлеры, логика, настройки и т.д.)
├── tests/            # Автотесты для проверки логики бота
├── requirements.txt  # Список Python-зависимостей
└── .gitignore
```

Внутри директории `bot/` обычно располагаются:

- модуль(и) инициализации бота (регистрация команд, запуск polling/webhook),
- хендлеры сообщений и команд,
- вспомогательные функции и утилиты (логирование, валидация, интеграции с внешними API),
- при необходимости — отдельный модуль с настройками/конфигом.

Конкретные имена файлов зависят от вашей реализации.

## Тестирование

Если у вас установлен **pytest** (чаще всего он подключается через `requirements.txt`), запустить тесты можно так:

```bash
pytest
```

Это поможет убедиться, что основные функции бота работают корректно после изменений.

## Как развивать проект дальше

Идеи для доработки бота:

- Добавить новые команды и сценарии диалога (например, тренировки по темам, карточки-слова дня).
- Реализовать хранение пользовательских настроек (уровень сложности, любимые темы и т.п.).
- Подключить базу данных (PostgreSQL, SQLite и др.) для истории диалогов и статистики.
- Вынести тексты сообщений и подсказок в отдельные конфигурационные файлы, чтобы было проще редактировать их без изменения кода.
- Добавить CI (GitHub Actions) для автозапуска тестов при каждом коммите.
